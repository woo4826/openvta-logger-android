# OpenVTA Logger User Share Guide

The text below can be sent to testers or friends.

```text
OpenVTA Logger 설치 및 사용 안내

1. 기존 앱 삭제
- 예전에 받은 VTA Logger / OpenVTA Logger 테스트 앱이 있으면 먼저 삭제해 주세요.
- 기존 앱에 필요한 기록 파일이 있으면 삭제 전에 Sessions 화면에서 VTA 또는 ZIP으로 먼저 공유/백업해 주세요.

2. 새 APK 다운로드
현재 공개 GitHub Release는 v0.0.3입니다. OpenVTA Live 기능 검증용 v0.0.4
APK는 배포 담당자가 별도로 제공한 링크와 SHA-256을 사용해 주세요.

https://github.com/woo4826/openvta-logger-android/releases/download/v0.0.3/OpenVTA-Logger-0.0.3-release.apk

3. 설치
- 위 링크를 Android 폰에서 열고 APK를 다운로드합니다.
- “알 수 없는 앱 설치” 허용이 뜨면 현재 브라우저/파일 앱에 대해 허용합니다.
- 설치 후 OpenVTA Logger를 실행합니다.
- 위치 권한과 알림 권한을 허용합니다.

4. 기본 사용
- Dashboard에서 Start를 누르면 기록이 시작됩니다.
- 기록 중에는 Live 화면에서 현재 위치, 이동 경로, GPS 상태를 볼 수 있습니다.
- Stop을 누르면 기록이 종료됩니다.
- Sessions 화면에서 기록된 VTA 파일을 확인할 수 있습니다.
- Zip 버튼으로 ZIP 파일을 만들 수 있습니다.
- Share VTA / Share ZIP 버튼으로 파일을 카톡, 메일, 드라이브 등으로 내보낼 수 있습니다.

5. IMU/GPS 보간 기능 설정
- Settings 화면으로 이동합니다.
- IMU / GPS enhancement preset 항목에서 원하는 모드를 선택합니다.
- 처음 테스트할 때는 Raw GPS를 추천합니다.

모드별 설명

1. Raw GPS
- GPS에서 들어온 원본 위치만 사용합니다.
- 가장 안정적이고 해석이 쉽습니다.
- GPS가 1초마다 들어오면 지도와 로그도 기본적으로 1Hz 기준입니다.
- 추천 상황:
  - 원본 데이터가 가장 중요한 경우
  - 기존 VTA 분석 도구와 호환성을 우선할 때
  - 테스트/비교 기준 데이터가 필요할 때
- Export:
  - `$` = 원본 GPS
  - `@` 보강 위치는 거의 없거나 최소화됩니다.

2. Linear 5Hz
- GPS 지점과 다음 GPS 지점 사이를 직선으로 나눠서 보간합니다.
- 예를 들어 GPS가 1Hz라면 그 사이를 5Hz처럼 촘촘하게 표시합니다.
- 장점:
  - 경로가 덜 끊겨 보입니다.
  - 데이터가 단순해서 이해하기 쉽습니다.
  - 일정하게 이동하는 구간에서 무난합니다.
- 한계:
  - 실제로 굽은 길을 갔어도 두 GPS 점 사이를 직선으로 연결합니다.
  - 실제 GPS 정확도가 좋아지는 것은 아닙니다.
- 추천 상황:
  - Live 지도 경로를 부드럽게 보고 싶을 때
  - 가장 보수적인 보간을 쓰고 싶을 때

3. Hermite 10Hz
- GPS 지점 사이를 속도/방향 흐름을 고려해 곡선으로 보간합니다.
- Linear보다 경로가 더 자연스럽게 이어집니다.
- 장점:
  - 경로 표시가 부드럽습니다.
  - 급격히 꺾이지 않고 자연스러운 곡선 형태가 됩니다.
  - 저장된 세션을 시각화할 때 보기 좋습니다.
- 한계:
  - 곡선은 추정값입니다.
  - GPS 오차가 큰 구간에서는 실제 경로와 다를 수 있습니다.
- 추천 상황:
  - 지도에서 경로를 자연스럽게 보고 싶을 때
  - 저장된 로그를 리뷰할 때

4. IMU heading 10Hz
- GPS 보간에 휴대폰 센서의 방향 정보를 일부 반영합니다.
- 이동 방향 변화가 있으면 보강 위치에 반영하려고 합니다.
- 장점:
  - 단순 직선 보간보다 방향 변화 표현이 나을 수 있습니다.
  - 회전/방향 변화가 있는 구간에서 더 그럴듯한 trace가 나올 수 있습니다.
- 한계:
  - 휴대폰 방향이 실제 진행 방향과 다르면 오차가 생깁니다.
  - 폰을 손에 들거나 거치 방향이 바뀌면 heading 값이 흔들릴 수 있습니다.
  - 센서 노이즈 영향을 받습니다.
- 추천 상황:
  - 폰이 차량/장비에 고정되어 있을 때
  - 방향 변화까지 실험적으로 보고 싶을 때

5. Bounded IMU dead reckoning 10Hz
- GPS 사이의 짧은 구간을 IMU 센서로 보강해 추정 위치를 만듭니다.
- 단, 오차가 너무 커지지 않도록 제한을 둔 bounded 방식입니다.
- 가장 적극적인 보강 모드입니다.
- 장점:
  - GPS 업데이트 사이의 움직임을 더 촘촘하게 추정합니다.
  - 짧은 GPS 공백이나 저속/회전 구간에서 더 많은 trace 후보를 볼 수 있습니다.
  - 원본 GPS와 별도로 보강 trace를 비교하기 좋습니다.
- 한계:
  - IMU dead reckoning은 시간이 지날수록 오차가 누적됩니다.
  - 폰이 흔들리거나 방향이 바뀌면 추정 위치가 틀어질 수 있습니다.
  - 실제 GPS 정확도가 좋아지는 것은 아닙니다.
  - 실험/분석용으로 보는 것이 좋습니다.
- 추천 상황:
  - GPS 원본과 IMU 보강값을 비교하고 싶을 때
  - 센서 기반 위치 보강 실험을 할 때
  - 폰이 안정적으로 고정된 상태에서 테스트할 때

6. Export 데이터 구분
OpenVTA Logger는 원본 GPS와 보강 데이터를 분리해서 저장합니다.

- `$` 줄: Android GPS 원본 위치
- `@` 줄: 선택한 모드로 생성된 보강/추정 위치
- `#` 줄: 센서 데이터

중요:
- 실제 측정값은 `$`를 기준으로 보면 됩니다.
- `@`는 보간 또는 IMU 기반 추정값입니다.
- 분석할 때는 `$`와 `@`를 섞어서 보지 말고 필요에 따라 분리해서 비교하는 것이 좋습니다.

추천 사용 순서:
1. 처음 테스트: Raw GPS
2. 지도 경로를 부드럽게 보기: Linear 5Hz
3. 보기 좋은 곡선 경로: Hermite 10Hz
4. 폰을 차량에 고정하고 방향 보강 테스트: IMU heading 10Hz
5. IMU 기반 위치 보강 실험: Bounded IMU dead reckoning 10Hz
```
